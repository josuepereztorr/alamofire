Connect to External API

Note: For the simplicity of this tutorial the API key will not be stored on an environment property such as a .plist.

Get API Key

Navigate to https://www.auto.dev and click on Sign In. From there either create a new account or sign in using a social login.

From the home page, click on the Vin Decode card which should navigate you to the Vin Decode page. From there click on Get started with VIN-Decode.

On the page there will be a section that will provide an example request. The image below shows an example of a request with an example vin (red underline) and your API key (I have blocked mine for security). You can test this by clicking on the link, which should return a JSON response to the test vin on the query.

Update HelperMethods.swift and Add the Alamofire Package

On Xcode, navigate to the HelperMethods.swift file and create a new let property called key. This property will store your API Key as a string which will be interpolated on the url property as seen in the image below. Add your API Key to the key property.

Finally, you will need to add the Alamofire package to your project. To do this right click on the root folder and select Add Package.... This should open a small window like the image below.

On the search bar, paste the GitHub repository link: https://github.com/Alamofire, then click Add Package. This should open a small window to fetch the package.

Now Xcode will fetch the package from the GitHub URL provided which it may take 1-2 minutes to complete. Once complete it will ask you to verify that you want to add this package. Click Add Package.

If the Alamofire package was successfuly added it should now appear under Package Dependencies as shown in the image below.

In order to use Alamofire, you must add an import statement on HelperMethods.swift. Add import Alamofire below the Foundation framework import as shown below.
