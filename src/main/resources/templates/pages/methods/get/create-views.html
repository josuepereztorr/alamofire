Create Views

Note: This tutorial will not handle optional errors and will be doing force unwrapping without catching.

Create Views Directory and SwiftUI Files

On the file explorer (left side of Xcode), click on the root directory vindecoder, then right click and select New Group. Create a new group called Views.

On the Views group, create a new file by right-clicking the Views group and selecting New File.... Then under User Interface select SwiftUI View and click Next. Name it InputView and click Create. This will create a file named InputView.swift

Use the same steps above to create OutputView.swift, ActionButtonView.swift, and ResultFieldView.swift. All the files under the Views group.

The root directory should now look like the image below.

Create Views

For InputView.swift we will need two bindings (to make them reactive) properties for the vin number and the vehicle of type Vehicle object (created in the Models step). Our view will be made of a VStack with a Text(), TextField() and ActionButtonView() views. I have also added some visual details such as font size and styling.

For OutputView.swift we will need only one binding property for the vehicle object (created in the Models step). Our view will be made of a VStack with a Text() and another VStack() containing the vehicle properties. The ResultFieldView view will take in arguments from the vehicle property and output them the view.

For FieldResultView.swift we will need a state binding property for the title placeholder and result. Our view will be made of a HStack with a two Text() views containing the title of the property the decoded vin property.

For ActionButtonView.swift we will need three binding properties for the vin, vehicle, and opacity of the error message. Our view will be made of a VStack with a Button() and a Text() view. When the button is clicked the logic of calling the external API will be run. If there is no vin entered the opacity of the error message will be to 1 but, if the decode button is clicked again with a vin number it will succeed.

For ContentView.swift we will need two binding properties for the vin and vehicle. Our view will be made of a VStack with a Text() view for the title of the app and another VStack() view containing our InputView and OutputView(). When the button is clicked the logic will be run and validated sending the data to the OutputView.
